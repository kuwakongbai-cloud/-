## 二、数据结构

### `Data_Structure`（定义在 Data.h）

每条记录存储 **一名学生在一门课程上的成绩**：

| 字段           | 类型        | 说明                          |
|----------------|-------------|-------------------------------|
| `name`         | char[30]    | 学生姓名                      |
| `score`        | int         | 该门课程的考试成绩（整数分）  |
| `project`      | char[6]     | 科目名称（如"数学""英语"）   |
| `student_id`   | char[30]    | 学号                          |
| `student_number` | int       | 序号（预留字段）              |
| `method`       | char[20]    | 查询/排序方法标识             |

> **注意**：一个学生如果有 n 门课，数组中就会对应 n 条独立的记录，按学号关联。这是该系统的核心设计特点。

## 三、函数原理详解

### 3.1 `Input_Data()` — 录入学生数据

**所在文件**：`Data.c`

**功能**：从标准输入读取 count 条学生成绩记录，存入数组。

**参数**：
- `data` — 数据数组起始位置（调用者通过偏移量指定写入位置）
- `count` — 要录入的记录条数

**实现原理**：
1. 循环 count 次，每次提示用户输入 4 个字段：`姓名 成绩 科目 学号`
2. 使用 `scanf()` 以空格分隔依次读取
3. 将每条记录的 `method` 字段初始化为空字符串

---

### 3.2 `Search()` — 通用查询函数

**所在文件**：`serchsort.c`

**功能**：在数据数组中按指定字段匹配关键字，返回所有匹配记录的索引数组。

**参数**：
- `data` — 数据数组
- `count` — 记录总数
- `method` — 查询字段类型：`"name"`（姓名）/ `"id"`（学号）/ `"score"`（成绩）/ `"project"`（科目）
- `key` — 查询关键字
- `result_count` — 输出参数，匹配到的记录数量

**返回值**：动态分配的 `int*` 数组（调用者须 `free()`），无匹配返回 `NULL`。

**算法原理**（两次遍历法）：
1. **第一遍遍历**：统计满足条件的记录数 `matched`
2. **动态分配**：`malloc(sizeof(int) * matched)` 精确分配结果数组
3. **第二遍遍历**：将匹配记录的下标填入结果数组

**为什么用两次遍历**：避免先分配过大的数组浪费内存，也不需要 realloc 动态扩容。适合数据量稳定的场景。

**匹配逻辑**：
- `name` / `id` / `project`：`strcmp()` 字符串精确匹配
- `score`：先用 `atoi()` 将 key 转为整数，再比较数值相等

---

### 3.3 `Sort()` — 排序分发函数

**所在文件**：`serchsort.c`

**功能**：根据 method 参数选择不同的排序策略，对数据数组进行原地排序（直接修改原数组顺序）。

**参数**：
- `data` — 数据数组（原地修改）
- `count` — 记录总数
- `method` — 排序方式（见下表）

| method          | 排序依据           | 顺序     | 比较方式             |
|-----------------|--------------------|----------|----------------------|
| `low-high`      | 成绩 (score)       | 升序     | `>` 数值比较         |
| `high-low`      | 成绩 (score)       | 降序     | `<` 数值比较         |
| `id_small-big`  | 学号 (student_id)  | 升序     | `strcmp() > 0`       |
| `dictionary`    | 姓名 (name)        | 升序     | `strcmp() > 0`       |

**算法原理**：冒泡排序（Bubble Sort）
- 外层循环控制轮数（n-1 轮）
- 内层循环进行相邻比较和交换
- 每轮结束后，当前未排序部分的最大（或最小）元素"冒泡"到正确位置
- 时间复杂度 O(n²)，实现简单，适合小型数据集

**交换方式**：直接结构体赋值 `Data_Structure temp = data[j]; data[j] = data[j+1]; data[j+1] = temp;`

---

### 3.4 `Lookup()` — 查找学生排名与各科成绩

**所在文件**：`serchsort.c`

**功能**：按姓名或学号查找学生，输出该学生所有科目的成绩明细。

**参数**：
- `data` — 数据数组
- `count` — 记录总数
- `method` — 查询方式：`"name"` 或 `"id"`
- `key` — 匹配关键字

**算法原理**（三步法）：

**第一步 — 学生去重与成绩汇总**
- 遍历所有记录，按学号去重
- 使用局部二维数组 `subjects[MAX_UNIQUE][20][6]` 和 `scores[MAX_UNIQUE][20]` 存储每个学生最多 20 门课的成绩
- 新学号 -> 添加到列表；已有学号 -> 追加科目成绩

**第二步 — 查找目标学生**
- 在去重后的学生列表中，按 method+key 线性查找目标

**第三步 — 输出结果**
- 以带边框的表格形式（Unicode 制表符）显示姓名、学号、各科成绩

---

### 3.5 `CourseStatistics()` — 课程成绩统计分析

**所在文件**：`serchsort.c`

**功能**：按课程分组，统计每门课程的成绩分布。

**参数**：
- `data` — 数据数组
- `count` — 记录总数

**算法原理**（两步法）：

**第一步 — 提取不重复课程**
- 遍历记录，使用 `courses[][]` 数组收集所有唯一的 `project` 值（最多 50 门课）

**第二步 — 逐门课程统计**
- 对每门课程再次遍历全数组，统计：
  - **学生总数**：该课程出现的总次数
  - **总分**：所有成绩累加
  - **各等级人数**：
    | 等级   | 分数区间 | 变量名     |
    |--------|----------|------------|
    | 优秀   | 90-100   | excellent  |
    | 良好   | 80-89    | good       |
    | 中等   | 70-79    | medium     |
    | 及格   | 60-69    | pass       |
    | 不及格 | 0-59     | fail       |
- 计算平均分及各等级百分比（`人数 / 总数 × 100%`）
- 以多层边框表格输出

**注意事项**：
- 一个学生在同一门课上有多次记录？会重复计入（系统按记录行统计，而非按人）
- `continue` 跳过不匹配课程，保证单次循环只处理目标课程

---

### 3.6 `StudentStatistics()` — 学生总分与平均分统计

**所在文件**：`serchsort.c`

**功能**：按学生（学号）分组，计算每位学生的总分和平均分，按总分降序排名输出。

**参数**：
- `data` — 数据数组
- `count` — 记录总数

**算法原理**（三步法）：

**第一步 — 学生去重与分数累加**
- 遍历记录，按学号去重
- 对每个学生同时维护：学号、姓名、选课数、总分
- 新学号 -> 初始化记录；已有学号 -> 选课数 +1、总分累加

**第二步 — 按总分降序排序**
- 使用冒泡排序按总分从高到低排列
- 交换时同步交换学生的**所有属性**（总分、选课数、学号、姓名），保持数据一致性

**第三步 — 输出排名表**
- 表头含排名、学号、姓名、总分、平均分
- 平均分 = 总分 / 选课数（`float` 除法）
- 以边框表格输出排名结果

---

### 3.7 `main()` — 主程序入口

**所在文件**：`main.c`

**功能**：运行主菜单循环，根据用户选择分派到各功能模块。

**算法原理**：
1. 初始化 `Data_Structure data[MAX_RECORDS]` 数组，`count = 0`
2. 进入 `while(1)` 无限循环
3. 打印主菜单（Unicode 边框装饰）
4. `scanf()` 读取用户选项，`switch-case` 分派
5. 每个 case 在操作前检查 `count == 0`（无数据时阻止操作）
6. 录入时检查数量合法性：`n > 0 && count + n <= MAX_RECORDS`

**菜单项对应关系**：

| 选项 | 功能               | 调用函数              |
|------|--------------------|-----------------------|
| 1    | 录入学生数据       | `Input_Data()`        |
| 2    | 查询学生           | `Search()`            |
| 3    | 排序数据           | `Sort()`              |
| 4    | 查找排名与各科成绩 | `Lookup()`            |
| 5    | 课程成绩统计       | `CourseStatistics()`  |
| 6    | 学生总分与平均分   | `StudentStatistics()` |
| 7    | 显示全部数据       | 直接遍历数组          |
| 0    | 退出系统           | `return 0`            |

---

### 3.8 `sort.c` — 排序转发模块

**所在文件**：`sort.c`

**功能**：纯粹的委托转发层。仅通过 `extern` 声明引用 `serchsort.c` 中的 `Sort()` 函数，本身不包含任何逻辑。

**设计意图**：可能最初规划将排序拆分到独立文件，最终实现在 `serchsort.c` 中，本文件作为模块入口保留。

---

### 3.9 `Serch()` 和 `LookupEntry()` — 查询包装模块

**所在文件**：`Serch.c`

**功能**：预留的查询入口包装层。

- **`Serch()`** — 函数体目前为空，为预留接口。通过 extern 声明引用 `Search()`。
- **`LookupEntry()`** — 对 `Lookup()` 的简单包装，通过 extern 声明引入并在内部调用。

**设计意图**：与 `sort.c` 类似，作为模块化入口层。当前实际调用在 `main.c` 中直接完成。
